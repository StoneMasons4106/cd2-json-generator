Enemy Caps:
Enemy Caps

Max Active Critters: Critters determines the maximum number of non-hostile creatures that can exist at once. This will not scale with player count.

Max Active Swarmers: Swarmers determines the maximum number of swarmer-sized enemies that can exist at once. This includes enemies such as Glyphid Swarmers, Naedocytes, and Shredders. If you want one consistent value, only use the 1 player scaling. If you use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Max Active Enemies: Enemies determines the maximum number of the rest of the enemies on the map that can exist at once. If it doesn't die in a pickaxe swing, it's likely in this pool. If you want one consistent value, only use the 1 player scaling. If you use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Resupply Cost:
Resupply Cost

If you would like a static, uniform resupply cost, enter a value in the one player scaling and move on. If you would like to scale resupply cost by player count, enter values in the 2, 3, and 4 player spots respectively.

If you would like to mutate your resupply cost, and take into account the number of resupplies called, select 'IfFloat' under the mutator dropdown. Then select 'ResuppliesCalled' in the 'Mutate By' dropdown. Then, enter the number of resupplies you'd like your condition to take effect on. In the 'Then' field, enter the cost when the number of resupplies you defined is met. Under the 'Else' field, enter the cost whenever the number of resupplies you defined is not met.
