Enemy Caps:
Enemy Caps

Max Active Critters: Critters determines the maximum number of non-hostile creatures that can exist at once. This will not scale with player count.

Max Active Swarmers: Swarmers determines the maximum number of swarmer-sized enemies that can exist at once. This includes enemies such as Glyphid Swarmers, Naedocytes, and Shredders. If you want one consistent value, only use the 1 player scaling. If you use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Max Active Enemies: Enemies determines the maximum number of the rest of the enemies on the map that can exist at once. If it doesn't die in a pickaxe swing, it's likely in this pool. If you want one consistent value, only use the 1 player scaling. If you use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Resupply Cost:
Resupply Cost

If you would like a static, uniform resupply cost, enter a value in the one player scaling and move on. If you would like to scale resupply cost by player count, enter values in the 2, 3, and 4 player spots respectively.

If you would like to mutate your resupply cost, and take into account the number of resupplies called, select 'IfFloat' under the mutator dropdown. Then select 'ResuppliesCalled' in the 'Mutate By' dropdown. Then, enter the number of resupplies you'd like your condition to take effect on. In the 'Then' field, enter the cost when the number of resupplies you defined is met. Under the 'Else' field, enter the cost whenever the number of resupplies you defined is not met.

Difficulty Setting:
Difficulty Setting

Extra Large Enemy Damage Resistance: The variant without a letter suffix affects the damage of the 3 dreadnought variants. B affects Korlok Tyrant Weeds, C affects the OMEN tower, D affects Elite enemies. If you would like a static value regardless of player count, please just use the one player scaling option. If you want to scale by player count and use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Enemy Damage Resistance: This field determines the damage resistance of most larger enemies in the game. This includes creatures such as Praetorians, Oppressors, Menaces, Goo Bombers, and Bulk Detonators, among others. If you would like a static value regardless of player count, please just use the one player scaling option. If you want to scale by player count and use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Small Enemy Damage Resistance: This field determines the damage resistance of enemies not covered by the other two resistance fields. This includes creatures such as Grunts and Grunt veterans, Mactera and Mactera veterans, and swarmers, among others. If you would like a static value regardless of player count, please just use the one player scaling option. If you want to scale by player count and use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Enemy Damage Modifier: This field determines the amount of damage enemies will deal with ALL attacks. If you would like a static value regardless of player count, please just use the one player scaling option. If you want to scale by player count and use only up until the 2 or 3 player scaling, everything with 2+ and 3+ players will have the same scaling, respectively.

Encounter Difficulty:
Encounter Difficulty

This field determines how many enemies will spawn as “Encounter” spawns. An Encounter spawn spawns when a player comes within a certain distance of an unexplored room in a cave. These spawns will usually include Grunt packs, Praetorians, Mactera groups, etc. These spawns DO NOT include stationary enemies, which are: Spitballers, Brood Nexii, Breeders and Cave Leeches. These spawns DO NOT include special event enemies such as Crassus Detonators, Huuli Hoarders, or Prospectors. These spawns will spawn un-aggroed, meaning a player will have to approach them or fire at them before they will attempt to attack. Each bin can be weighted between 0 and 1, 0 being 0 percent, 1 being 100 percent. Then you can assign a minimum number of points to assign to the bin, as well as a maximum weight.
